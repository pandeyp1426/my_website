RankMyStocks

Full stack stock ranking and portfolio tracker built with a React single page app connected to a Flask API for market data, news, and persistence.

React 19 + Vite Flask + MySQL Auth0 Redux Toolkit
Live Demo GitHub Repo

Overview

RankMyStocks helps users build ranked portfolios quickly with a guided questionnaire and a clean, data-driven UI.

  • Full stack SPA with React routes and a Flask REST API.
  • MySQL stores users, portfolios, transactions, and snapshots.
  • Market data and news pulled on demand with graceful fallbacks.

User Features

  • Auth0 login with saved portfolios tied to each user.
  • Questionnaire-driven stock ranking to build a portfolio fast.
  • Rename or delete portfolios and track changes over time.
  • Custom SVG chart with line or candlestick toggle.
  • Market news feed plus a daily digest.

How It Works

  1. React routes: Home, Questionnaire, My Portfolios, Rankings, News. Auth0 handles login; Redux stores user and questionnaire state.
  2. Starting a portfolio calls /api/init. Flask filters tickers (ticker_list.csv + DB metrics) and stores the list in session.
  3. /api/next serves stock pairs; /api/get-stock-data fills in prices and short summaries.
  4. Picks are tracked via /api/pick. After rounds, the frontend saves to MySQL through /api/portfolios. Users can rename or delete portfolios.
  5. Portfolio chart from /api/portfolio-chart. The API aggregates Yahoo Finance candles across holdings; the UI renders a custom SVG chart with line or candlestick toggle.
  6. News + digest from /api/market-news and /api/daily-digest. Pulls from external news APIs and optionally summarizes using OpenAI via LangChain when keys are configured.

Tech Stack

  • Frontend: React 19 + Vite, React Router, Redux Toolkit, Auth0 React SDK, Tailwind CSS + custom CSS, Fetch/Axios.
  • Backend: Flask, flask-cors, python-dotenv, mysql-connector, requests, pandas, yfinance.
  • Data: MySQL (users, portfolios, transactions, snapshots) with ticker_list.csv as the universe.
  • Optional: LangChain + OpenAI for summaries.

Technical Highlights

  • Session-based questionnaire queue and state management.
  • REST API design with validation and graceful error handling.
  • MySQL persistence for portfolios and historical snapshots.
  • Custom SVG chart rendering with line/candlestick toggle.
  • Optional LLM summaries and news digest gated behind environment keys.

Future Improvements

  • Automated API and UI tests.
  • Better caching and cache invalidation for market data.
  • Background jobs for daily snapshots.
  • Rate limit backoff and pagination for news APIs.
  • Logging and monitoring.

Configuration notes for running locally or deploying a custom instance.

  • API base uses VITE_API_URL (defaults to /api).
  • Auth0 config via VITE_AUTH0_* or REACT_APP_AUTH0_*.
  • Optional news APIs (Marketaux, Alpha Vantage, Finnhub, FMP) and OpenAI keys are read from .env.
